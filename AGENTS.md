# STATUS SEO repository guidance

This repository contains the STATUS dental-clinic website and its client-specific SEO operating state.

## Required context

- Before SEO work, read `.seo/CHATGPT_PROJECT_START.md`, `.seo/project.json`, `.seo/intake.md`, `.seo/backlog.csv`, `.seo/page-map.csv` and `.seo/change-log.md`.
- Use the reusable SEO Site Operator skills when installed. Route full-program work through `seo-command-center` and use the specialist skills for technical, content, local, AI-discovery, authority and measurement tasks.
- Continue the existing implementation and draft PR #1 instead of starting a duplicate audit, branch or orthodontics page.

## Working rules

- Treat user-confirmed facts, public-source observations, third-party metrics and inference as different evidence types.
- Never invent services, doctors, credentials, reviews, cases, prices, rankings, traffic, backlinks or medical outcomes.
- Keep medical claims evidence-based and require clinic/doctor review where noted in `.seo/`.
- Work in small reviewable batches. Update `.seo/backlog.csv` and `.seo/change-log.md` for every material batch.
- Run relevant source, link, metadata, JSON-LD, sitemap, JavaScript and diff checks before handoff.
- Never store credentials, analytics secrets or Telegram bot tokens in browser JavaScript, chat output, `.seo/` or version control.
- Do not merge, deploy, submit sitemaps, modify Google Business Profile or change external accounts without explicit authorization.

## Current conversion constraint

Website forms do not have a verified delivery endpoint. Telegram is planned through a secure server-side relay. Never show or report a successful submission unless the server confirms delivery.

## Handoff

Lead with the result. Report the branch/PR, validation performed, unresolved evidence, blockers and the smallest useful next decision. Reply in the user's language.
